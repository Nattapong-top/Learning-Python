🧭 ROADMAP: Programming Theory for Real Dev (Python Track)
🔰 Phase 1: Core Logic Foundation (ตอนนี้กำลังอยู่ตรงนี้)

เป้าหมาย: อ่านโจทย์แล้ว “เห็นวิธีคิด” ทันที

📘 ทฤษฎีที่ต้องเข้าใจ

Input / Output แยกจากกันให้ขาด

Loop ซ้อน (nested while / for)

การตรวจข้อมูลก่อนใช้ (isdigit, type check)

ไม่แก้ input โดยไม่จำเป็น

ฟังก์ชัน = หน่วยความคิด

🧪 ชุดโจทย์ฝึก (ไม่เฉลยอัตโนมัติ)

รับข้อมูลหลายบรรทัด จนเจอ END
👉 นับ / หาค่า / กรองเฉพาะที่ถูกต้อง

แยก logic ออกเป็นฟังก์ชันย่อย

ห้ามใช้ max() / min() (บังคับคิดเอง)

nested loop ที่ ต้องมีเหตุผล ไม่ใช่ซ้อนมั่ว

📌 กติกา Phase นี้

เขียน วิธีคิด logic 5 ข้อ ก่อนโค้ดทุกครั้ง

โค้ดผ่าน แต่ logic อธิบายไม่ได้ = ยังไม่ผ่าน

⚙️ Phase 2: Data Thinking & Pattern

เป้าหมาย: มองข้อมูลออกว่า “ควรเก็บแบบไหน”

📘 ทฤษฎี

list vs dict vs set

การวนข้อมูล 2 มิติ

การสะสมค่า (accumulator pattern)

ความสัมพันธ์ของ index

🧪 โจทย์

นับความถี่ตัวเลข / ตัวอักษร

รวมข้อมูลจากหลาย list

หาค่าซ้ำ / ค่าที่หายไป

แปลงข้อมูลจากรูปแบบหนึ่ง → อีกรูปแบบ

📌 สิ่งที่ต้องอธิบายได้

ทำไมใช้ dict ไม่ใช้ list

ถ้า input โตขึ้น โค้ดจะพังตรงไหน

🧩 Phase 3: Function Design & Decomposition

เป้าหมาย: โค้ดอ่านเหมือนภาษาอังกฤษ

📘 ทฤษฎี

Single Responsibility

Function contract (รับอะไร คืนอะไร)

ไม่ print ใน function (return เท่านั้น)

Reuse ได้จริง

🧪 โจทย์

แยกโจทย์ใหญ่ → ฟังก์ชันย่อย 3–5 ตัว

เขียนเวอร์ชัน loop ปกติ + pythonic

แก้โจทย์เดิม แต่ห้ามแก้โค้ดเดิม (ต้องต่อยอด)

🧠 Phase 4: Algorithm Mindset (เบา ๆ แต่สำคัญ)

เป้าหมาย: รู้ว่าโค้ดแบบไหน “ควรใช้จริง”

📘 ทฤษฎี

เปรียบเทียบหลายวิธีแก้โจทย์เดียว

Time complexity แบบไม่ต้อง formal

เลี่ยงการ sort ถ้าไม่จำเป็น

คิด worst case เสมอ

🧪 โจทย์

หา min / max / closest pair

หาค่าที่ตรงเงื่อนไขเร็วที่สุด

แก้โจทย์เดียว 2 วิธี แล้วอธิบายข้อดีข้อเสีย

🏗️ Phase 5: Real-World Style Problems

เป้าหมาย: พร้อมต่อยอดไปโปรเจกต์

🧪 โจทย์แนวระบบ

อ่าน log

จัดการ asset / inventory

ตรวจข้อมูลผิดพลาด

สรุปรายงานจาก input ดิบ

📌 Phase นี้ = สะพานไปโปรเจกต์ทะเบียนสินทรัพย์ที่ป๋าอยากทำ 🔥
